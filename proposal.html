Problem Description

Question of interest?

The area of interest I'm focusing on for this project is literature. I'm an avid reader and writer so this area seems like the perfect place for me to build a tool that I would like to use. The problem that I'm focusing on is addressing current issues with Goodreads' user interface and experience.

What is the information technology problem?

Goodreads is the most well known service for cataloguing books for personal use. They allow users to sort books into bookshelves, explore books they might be interested in, and leave reviews. I want to address some well known UX issues that users have with the mobile application. I will be focusing specifically on the shelving concept, drawing inspiration from Spotify, Pinterest, etc.

What already exists?

Goodreads already exists on its own as an established application, however, I want to create a website that serves as a sort of proposal for how Goodreads could elevate a part of their user experience.

Proposed Application

Who are the users of this application?

This would be meant for current, new, or prespective Goodreads users who want a more modern experience for sorting books. I also want to make sure the books that are being reccommended are from all backgrounds so that people are able to get exposure to books written by BIPOC, female, LGBTQ+, and/or disabled authors, because the publishing world is not exempt from the systemic effects of racism, ableism, sexism, and intolerance of any kind.

Interactive Features

  1. Allow users to create a bookshelf in which they can add books. There will be various customization options for each bookshelf.
  2. Users can sort and filter through existing books that will be accessed through datasets pulled from Goodreads.
  3. Log into an account so that they can save their bookselhves and access them whenever.
  4. Add reviews to any book that is only for their consumption (i.e they see it when they shelf a book, no other users will see).

Data Sources

I will primarily be using Kaggle Datasets here, such as this dataset with Goodreads books or Goodreads dataset with links to the cover art. It is an option to use the Goodreads API, but I'm not completely confident that that is a good idea, since I will be working alone. I plan to discuss it further with my TA, though I can imagine I could achieve the goal of the website either using Kaggle scraped datasets or the Goodreads API. Also it is an option to use Amazon books data for more diverse data.

How will this solve my original problem?

This application will allow me to use creativity and user research to address concerns with Goodreads' existing feature of book shelving. I can also work to find data on books written by or featuring POC so that the application works to address issues around low visibility for BIPOC authors and encourages diversity of reading for users.
